位运算

发表于 2019-12-26 更新于 2020-10-24 分类于 算法 ， 基础算法 阅读次数： Valine：
本文字数： 742 字

位运算操作符介绍，源码、反码、补码相关，位运算常用操作

操作符

& 按位与：同1则1 其余0

| 按位或：有1则1 都0则0

^ 按位异或：同0 异1

~ 按位取反：0变1 1变0 （符号位不变）

<< 左移：左移n位 = * 2^n

>> 带符号右移 右移n位 = / 2^n （带符号右移 正数右移后高位补0 负数右移后高位补1）

>>> 无符号右移 右移n位 （无符号右移 正负数高位均补0）

源码、反码、补码

计算储存整型数时 最高位0代表正数 1代表负数 int 型可以储存 [-2^32,2^32-1] （c语言规定10000000000000000000000000000000为-2147483648的补码） 源码：二进制数 反码：二进制数各位取反 补码：二进制数各位取反后加1 注：在计算机底层没有减法操作，因此计算机用补码的方式储存负数 例：设x二进制数为 00000000000000000000000000000001 ~x为 11111111111111111111111111111110 又 -x = 0 - x 0的二进制数为 00000000000000000000000000000000 假设可以加一高位1 100000000000000000000000000000000 - x = ~x + 1 （超出范围的高位舍去）

常用的几个操作

1.求n的二进制数最后一位

n & 1

2.求n的二进制数第k位是多少

n >> k & 1
/* n的第k位指从右往左由0开始计位 */

3.lowbit (x) 操作 返回x的最后一位1 （100100 则返回 100）

x & -x
/* x & (~x + 1) */

假设 x 为 1010…10000 ~x = 0101…01111 ~x + 1 = 0101…10000 x & ~x+1 后 为 0000…10000
